    Introduction

    Welcome to the Gate API documantation! You can use Gate API to synchronize VES and Ocean.

    Authentication

    Gate uses API keys to allow access to the API. You cannot register a new Gate API key, it will be provided by someone in WHO. Gate expects for the API key to be included in all API requests to the server in a header that looks like the following:

    Authorization: "Bearer $TOKEN$"

    You must replace $TOKEN$ with your API key.

    Request Token

    curl -X POST -H "Content-Type: application/json" -H -d '{
  "client_id": "9f1b8b5649e62f1531e67f445b62103e0d91298c19491390a37bb9ef8db13a32",
  "client_secret": "653ce03911d988aaa6ad8ac7be292a39e7b0ac76ee9e3a8146d9c292b4919009",
  "grant_type": "client_credentials",
  "scope": "gate"
}' "http://localhost:3000/oauth/token"

    The above command returns JSON structured like this:

    HTTP Request

    POST http://ocean.vispera.co/oauth/token

    Query Parameters

    Parameter Description
    client_id Client ID that provided to you.
    client_secret Client secret that provided to you.
    grant_type Use “client_credentials” to get token.
    scope Scope of your token, in this case it is “gate”

    Revoke Token

    curl -X POST -H "Content-Type: application/json" -H "Cache-Control: no-cache" '{
    "token": "$TOKEN$"
}' "http://ocean.vispera.co/oauth/revoke"

    The above command does not return anything expect HTTP Status code 204.

    HTTP Request

    POST http://ocean.vispera.co/oauth/revoke

    Query Parameters

    Parameter Description
    token Token that you want to revoke.

    Update Specific Visit

    curl -X PUT -H "Authorization: Bearer $TOKEN$ "http://localhost:3000/api/gate/v1/visits/$ID$" -d '{
    "visit": {
        "status": "accomplished"
    }

    Update specific visit with ID.

    The above command does not return anything expect HTTP Status code 204.

    HTTP Request

    PUT http://localhost:3000/api/gate/v1/visits/$ID$

    URL Parameters

    Parameter Description
    ID The ID of the visit

    Query Parameters

    Parameter Description
    JSON Data Json data of visit (fields you want to change) 
    Visit data example:
{"download_completed": true || false, "visit": { "status": "failed" || "accomplished" }}

    Errors

    Do not forget to add your token in request head like this: Authorization: Bearer $TOKEN$

    The Shark API uses the following error codes:

    Error Code Meaning
    400 Bad Request – Your request sucks
    401 Unauthorized – Your API key is wrong
    403 Forbidden – The requested is hidden for administrators only
    404 Not Found – The specified resource could not be found
    405 Method Not Allowed – You tried to access a resource with an invalid method
    406 Not Acceptable – You requested a format that isn’t json
    410 Gone – The resource requested has been removed from our servers
    418 I’m a teapot
    429 Too Many Requests – You’re requesting too many resources! Slow down!
    500 Internal Server Error – We had a problem with our server. Try again later.
    503 Service Unavailable – We’re temporarially offline for maintanance. Please try again later.
